# TerraSoviet Data Rescue — Track 1: NLP & Document AI

Пайплайн для извлечения структурированных данных из отсканированных
советских геологических отчётов. Превращает сканы PDF в машиночитаемый JSON.

## Что делает

1. Берёт PDF-отчёты из папки `input/`
2. Распознаёт кириллический текст (OCR через Tesseract)
3. Парсит таблицы (каталоги координат, каталоги проявлений)
4. Извлекает геологические сущности (минералы, участки, координаты, содержания)
5. Сохраняет структурированный JSON в `output/`
6. Логирует всё, что не удалось обработать — ничего не теряется молча

## Подход

Решение спроектировано под критерии хакатона:
- **Масштабируемость**: обрабатывает ЛЮБОЕ число PDF в папке `input/` через цикл,
  без хардкода имён файлов. Добавь хоть 10 000 файлов — запустится так же.
- **Воспроизводимость**: пути относительные, зависимости в `requirements.txt`,
  запуск одной командой.
- **Надёжность**: каждый файл в try/except. Один битый скан не роняет весь пайплайн —
  он попадает в лог ошибок, остальные обрабатываются.

Принцип (совет ментора): лучше обработать 80% документов хорошо и честно отметить
остальные как необработанные, чем хардкодить правила под каждый редкий случай.

## Установка

```bash
# системная зависимость — Tesseract с русским языком
# Ubuntu/Debian:
sudo apt-get install tesseract-ocr tesseract-ocr-rus poppler-utils
# macOS:
brew install tesseract tesseract-lang poppler

# python-зависимости
pip install -r requirements.txt
```

## Запуск

```bash
# положи PDF-файлы в папку input/, затем:
python src/main.py

# или укажи свои папки:
python src/main.py --input путь/к/сканам --output путь/к/результату
```

## Структура проекта

```
track1/
├── input/              # сюда кладутся PDF-сканы
├── output/             # сюда пишется JSON-результат + лог ошибок
├── src/
│   ├── main.py         # точка входа, обходит все файлы
│   ├── ocr.py          # PDF → текст (Tesseract)
│   ├── parse_catalog.py# парсинг таблиц-каталогов
│   └── entities.py     # извлечение геологических сущностей
├── requirements.txt
└── README.md
```

## Формат выхода

Для каждого PDF создаётся `<имя>.json`:

```json
{
  "source_file": "каталог_координат_25834.pdf",
  "report_id": "25834",
  "raw_text": "...распознанный текст...",
  "coordinates": [
    {"id": "1", "name": "Ю.Жуманай", "lat_dms": "48°03'", "lon_dms": "71°16'"}
  ],
  "entities": {
    "minerals": ["золото", "серебро", "висмут"],
    "locations": ["Агадырский район", "Тологай"]
  }
}
```
