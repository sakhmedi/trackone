# Пример работы пайплайна

Эта папка — готовый пример «вход → выход», чтобы можно было увидеть результат
**не запуская** ничего. Всё здесь сгенерировано реальным прогоном `main.py`
на настоящем отчёте, а не написано вручную.

## Что лежит

Два примера из одного отчёта №25834 — они показывают **разные половины** пайплайна:

| Файл | Что это |
|------|---------|
| `каталог_координат_25834.pdf` | **вход** — отсканированный советский каталог координат (4 стр.) |
| `каталог_координат_25834.json` / `.md` | **выход** — извлечение **координат** участков (геосторона) |
| `протокол_25834.json` / `.md` | **выход** — извлечение **сущностей** (минералы, содержания г/т) из 12 стр. сплошного текста протокола |
| `all_coordinates.geojson` | **выход** — все валидные координаты одним файлом (открывается на [geojson.io](https://geojson.io)) |

> Сам `протокол_25834.pdf` (8 МБ) в репозиторий не кладётся, чтобы не раздувать его;
> лежащие здесь JSON/MD получены прогоном `main.py` на нём.

## Как это воспроизвести

Из корня репозитория:

```bash
python main.py --input examples --output examples
```

(уже обработанный файл будет пропущен — добавьте `--overwrite`, чтобы пересчитать).

## Почему оба примера помечены `needs_review`

Это честные примеры, а не «причёсанные». Оба документа OCR распознал со средней
уверенностью около **50%** (каталог — 49.7%, протокол — 50.8%), поэтому оба
автоматически помечены `"needs_review": true`.

Так и должно быть: это **объективное свойство датасета**, а не дефект пайплайна.
Сканы — машинопись начала 1970-х на выцветшей бумаге; даже человек читает их с
трудом, и ни один OCR не даст по ним «уверенных» 90%. Ценность решения как раз в
том, что оно:

- **не выдаёт ненадёжное за надёжное** — честно ставит флаг и среднюю уверенность,
  чтобы оператор знал, что проверить (совет ментора: «лучше честно отметить
  ненадёжное, чем хардкодить»);
- **всё равно извлекает структуру** из такого входа: из каталога — координаты
  участков, из протокола — минералы (золото, серебро, висмут, медь, цинк, кварц)
  и содержания в г/т. В выходе видны и реальные ошибки OCR, но они помечены
  (`valid: false`, сырые значения сохранены), а не выброшены молча.

То есть низкая уверенность здесь — это работающий индикатор качества, а не
провал распознавания.
