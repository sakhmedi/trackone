# Пример работы пайплайна

Эта папка — готовый пример «вход → выход», чтобы можно было увидеть результат
**не запуская** ничего. Всё здесь сгенерировано реальным прогоном `main.py`
на настоящем отчёте, а не написано вручную.

## Что лежит

| Файл | Что это |
|------|---------|
| `каталог_координат_25834.pdf` | **вход** — отсканированный советский каталог координат (4 стр.) |
| `каталог_координат_25834.json` | **выход** — структурированный JSON (координаты, сущности, мета OCR, полный текст) |
| `каталог_координат_25834.md` | **выход** — человекочитаемое саммари |
| `all_coordinates.geojson` | **выход** — все валидные координаты одним файлом (открывается на [geojson.io](https://geojson.io)) |

## Как это воспроизвести

Из корня репозитория:

```bash
python main.py --input examples --output examples
```

(уже обработанный файл будет пропущен — добавьте `--overwrite`, чтобы пересчитать).

## На что обратить внимание

Это честный пример, а не «причёсанный». OCR этого скана дал среднюю уверенность
**49.7%**, поэтому документ автоматически помечен `"needs_review": true`. В выходе
видны и реальные ошибки распознавания (например, «заладный» вместо «западный»,
номер участка `0`, одна явно искажённая долгота). Пайплайн **не скрывает** это:
такие данные помечаются, а не выбрасываются молча — ровно как требует совет ментора
(«лучше честно отметить ненадёжное, чем хардкодить»).
